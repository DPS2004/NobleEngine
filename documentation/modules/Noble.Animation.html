Noble.Animation

Animation states using a spritesheet/imagetable.

Ideal for use with playdate.graphics.sprite objects, but suitable for other uses as well.

Setup

Noble.Animation.new(__spritesheet)
Create a new animation "state machine".

Parameters

  • __spritesheet string
    The path to the bitmap spritesheet/imagetable asset.

Returns

    animation, a new animation object.
animation:addState(__name, __startFrame, __endFrame, __next)
Add an animation state. The first state added will be the default for this animation.

Parameters

  • __name string
    The name of the animation, this is also used as the key for the animation.
  • __startFrame int
    This is the first frame for this animation in the imagetable/spritesheet
  • __endFrame int
    This is the first frame for this animation in the imagetable/spritesheet
  • __next string
    By default, animation states will loop, but if you want to sequence an animation, enter the name of the next animation here.
animation:setState(__animationState[, __continuous=false])
Changes

Parameters

  • __animationState string or Noble.Animation
    The name of the animation to set. You can pass the name of the state, or the object itself.
  • __continuous bool = false (default)
    Set to true if your new state's frames line up with the previous one's, i.e.: two walk cycles but one is wearing a cute hat!

Usage

animation:setState("run")
animation:setState(animation.run)
animation:setState(animation.walkNoHat)
--
animation:setState(animation.walkYesHat, true)
animation:draw([__x=0[, __y=0]])
Draw the current frame.

Parameters

  • __x number = 0 (default)
  • __y number = 0 (default)

Usage

function MySprite:draw()
	animation:draw()
end
function MyScene:update()
	animation:draw(100,100)
end
